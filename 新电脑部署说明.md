# 项目部署说明（新电脑从零开始）

本文档用于说明如何在一台新的 Windows 电脑上，从零开始部署并运行本项目。文档按“先跑工程原型，再跑研究系统”的顺序编写。若你只需要演示后端和前端，可以先完成前半部分；若还需要跑论文中的多模型实验和 Streamlit 看板，再继续完成研究系统部分。

## 1. 项目结构说明

当前仓库实际上包含两套系统：

1. 工程原型
   - 后端：FastAPI
   - 数据库：MySQL 8（Docker）
   - 前端：Vue 3 + Element Plus + ECharts
2. 研究系统
   - 数据处理与实验：Pandas、statsmodels、XGBoost、Prophet、PyTorch、TFT、SHAP
   - 可视化：Streamlit + Plotly

建议把它理解成：

- `app/`：工程原型后端
- `frontend/`：工程原型前端
- `research_app/`：论文实验与研究展示系统
- `毕设数据统计.csv` / `毕设数据统计.xlsx`：基础数据文件

这两套系统在数据读取方式上也不完全相同：

1. 工程原型
   - 后端读取的是 MySQL 中的数据
   - 数据通常先通过导入接口写入数据库，再由 FastAPI 提供给前端页面
   - 可以理解为：`文件 -> MySQL -> FastAPI -> Vue`
2. 研究系统
   - 默认直接读取项目根目录下的本地数据文件
   - 当前主要使用的是 [毕设数据统计.csv](C:/Coding/260214_tourist-traffic/毕设数据统计.csv)
   - 可以理解为：`CSV -> pandas -> 模型实验 -> Streamlit`

因此需要特别注意：

- 启动工程原型时，数据库必须可用
- 启动研究系统时，不依赖 MySQL 也可以运行
- 两套系统展示的是同一个项目，但底层数据读取路径不同

本项目支持两种本地数据库部署方式：

1. Docker 版 MySQL 8
2. Windows 本机安装版 MySQL 8

如果你的新电脑已经装好了普通 Windows 版 MySQL 8，建议直接使用本机 MySQL，不必强制再装 Docker。下面两种方式任选其一即可。

## 5A. 启动 MySQL（Docker）

如果你准备使用 Docker 版 MySQL 8，项目根目录已经提供了 `docker-compose.yml`，直接执行：

```powershell
docker compose up -d
```

启动完成后，可使用以下命令查看状态：

```powershell
docker compose ps
```

正常情况下你会看到一个名为 `tourist_traffic_mysql` 的容器处于 `Up` 状态。

默认数据库连接信息如下：

- Host：`127.0.0.1`
- Port：`3306`
- Database：`tourist_traffic`
- User：`tourist`
- Password：`tourist`
- Root Password：`root`

如果 MySQL 启动异常，可以查看日志：

```powershell
docker compose logs --tail 100 mysql
```

## 5B. 启动 MySQL（Windows 本机安装版）

如果你使用的是普通 Windows 版本的 MySQL 8，请按下面步骤处理。

### 5B.1 确认 MySQL 服务已启动

可以先在 PowerShell 中查看服务：

```powershell
Get-Service | Where-Object { $_.Name -like 'MySQL*' }
```

常见服务名可能是：

- `MySQL80`
- `mysql`

如果服务未启动，可执行：

```powershell
Start-Service MySQL80
```

如果你的服务名不是 `MySQL80`，请改成你机器上的实际服务名。

### 5B.2 登录 MySQL

在 PowerShell 中执行：

```powershell
mysql -u root -p
```

然后输入你本机 MySQL 的 root 密码。

如果系统提示找不到 `mysql` 命令，说明 MySQL 的 `bin` 目录没有加入环境变量。常见路径例如：

```text
C:\Program Files\MySQL\MySQL Server 8.0\bin
```

可以临时切到该目录后再执行：

```powershell
cd "C:\Program Files\MySQL\MySQL Server 8.0\bin"
.\mysql.exe -u root -p
```

### 5B.3 创建数据库和业务用户

登录 MySQL 后执行：

```sql
CREATE DATABASE tourist_traffic CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'tourist'@'localhost' IDENTIFIED BY 'tourist';
GRANT ALL PRIVILEGES ON tourist_traffic.* TO 'tourist'@'localhost';
FLUSH PRIVILEGES;
```

如果你已经有现成的账号，也可以不创建 `tourist` 用户，但需要在 `.env` 中改成你自己的账号密码。

### 5B.4 修改 `.env` 数据库连接

如果使用 Windows 本机 MySQL，建议确认 `.env` 中数据库连接指向本机服务。通常配置应类似于：

```env
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=tourist_traffic
DB_USER=tourist
DB_PASSWORD=tourist
```

如果你使用 root 账号，也可以改成：

```env
DB_USER=root
DB_PASSWORD=你的root密码
```

### 5B.5 检查数据库是否可连接

你可以使用 Navicat、DBeaver 或 MySQL Workbench 检查是否能连上：

- Host：`127.0.0.1`
- Port：`3306`
- Database：`tourist_traffic`
- User：`tourist` 或 `root`
- Password：对应密码

如果能连通，说明数据库部分已经准备好。

## 6. 创建工程原型后端虚拟环境

如果新电脑上没有 `venv`，可在项目根目录创建：

```powershell
py -3.12 -m venv venv
```

激活虚拟环境：

```powershell
.\venv\Scripts\Activate.ps1
```

安装依赖：

```powershell
.\venv\Scripts\python.exe -m pip install --upgrade pip
.\venv\Scripts\python.exe -m pip install -r requirements.txt
```

说明：

- 当前项目已经把工程原型和研究系统的大部分依赖统一放进 [requirements.txt](C:/Coding/260214_tourist-traffic/requirements.txt)。
- 正常情况下，只执行一次 `pip install -r requirements.txt` 就足够。
- 如果你只想先跑工程原型，完成这一步后就已经可以启动后端。

## 7. 启动工程原型后端

在项目根目录执行：

```powershell
.\venv\Scripts\python.exe -m uvicorn app.main:app --reload
```

启动后可访问：

- API 首页：`http://127.0.0.1:8000`
- Swagger 文档：`http://127.0.0.1:8000/docs`

默认管理员账号通常为：

如果前端登录时报跨域错误，需要确认后端已经是当前仓库里的最新版本，因为项目里已加入 CORS 配置。

1. MySQL 已启动

3. 后端地址仍为 `http://127.0.0.1:8000`

## 15. 常见问题

### 15.1 Docker 没启动

现象：

- `docker compose up -d` 报错
- `docker compose ps` 无法执行

处理：

1. 确认 Docker Desktop 已启动
2. 等待 Docker 引擎完全就绪后再执行命令

如果你本来就使用 Windows 本机 MySQL，这一条可以直接跳过。

### 15.2 Windows 本机 MySQL 无法连接

现象：

- 后端启动时报数据库连接失败
- Navicat 或 Workbench 也连不上本地 MySQL

处理：

1. 确认 MySQL 服务已启动：

```powershell
Get-Service MySQL80
```

2. 若未启动，执行：

```powershell
Start-Service MySQL80
```

3. 确认 `.env` 中的 `DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD` 配置正确
4. 确认本地 MySQL 监听端口确实是 `3306`
5. 确认你创建过 `tourist_traffic` 数据库

### 15.3 前端登录时报 405 或跨域错误

现象：

- 登录接口请求失败
- 浏览器控制台出现 `OPTIONS` 请求错误

处理：

1. 确认后端是当前仓库的最新代码
2. 重启 `uvicorn`
3. 确认访问地址为 `http://127.0.0.1:3000`

### 15.4 Streamlit 页面启动后报 Torch 或 watcher 错误

本项目已在 `.streamlit/config.toml` 中关闭了相关文件监听，正常情况下不会影响使用。如果仍然报错，优先确认：

1. 你启动的是 `research_app\app.py`
2. 使用的是 `venv312`
3. `torch`、`lightning`、`pytorch-forecasting` 已安装

### 15.5 Streamlit 启动时报 `WinError 10013`

现象：

- 启动 Streamlit 报：
  `PermissionError: [WinError 10013]`
- 甚至执行 `python -m http.server 8512` 也报同样错误

这通常不是项目代码问题，而是本机端口被 Windows 排除或限制了。某些机器会把 `8501` 附近这一段端口列入系统排除范围，因此 Streamlit 默认端口无法绑定。

可以先执行：

```powershell
netsh interface ipv4 show excludedportrange protocol=tcp
```

如果结果中包含 `8501`、`8502`、`8512` 等端口所在区间，就不要继续使用这些端口，直接换一个不冲突的端口，例如：

```powershell
.\venv312\Scripts\python.exe -m streamlit run research_app\app.py --server.address 127.0.0.1 --server.port 18080
```

也可以先用下面命令测试这个端口是否能监听：

```powershell
python -m http.server 18080
```

如果 `18080` 能正常启动，就说明 Python 环境没有问题，只是默认端口段被系统占用了。

### 15.6 Prophet 报 `tbb.dll` 缺失

处理：

```powershell
.\venv312\Scripts\python.exe -m pip install tbb
```

安装后重启 Streamlit。

### 15.7 缺少 Python 模块

如果启动时报缺模块，优先重新安装依赖：

```powershell
.\venv\Scripts\python.exe -m pip install -r requirements.txt
.\venv312\Scripts\python.exe -m pip install -r requirements.txt
```

如果仍然提示某个研究系统依赖缺失，再单独补装对应包，例如：

```powershell
.\venv312\Scripts\python.exe -m pip install torch lightning pytorch-forecasting tbb
```

## 16. 部署完成后的推荐操作

新电脑完成部署后，建议立即做以下几件事：

1. 先确认三套入口都能打开：
   - 后端
   - 前端
   - Streamlit
2. 先不要修改数据库账号和接口地址，确保基础链路跑通
3. 登录前端导入一次数据文件，确认数据库可写
4. 在研究系统中切换一次 `7 / 14 / 30` horizon，确认模型对比页能正常加载
5. 若用于答辩演示，建议提前把需要展示的页面都跑一遍
