## 荔波县旅游客流分析与预测系统开发文档

### 一、选题意义与项目定位

本项目以荔波县旅游客流为研究对象，围绕“多源数据驱动的旅游客流影响因素分析与预测”展开。项目目标不是只做一个简单的预测接口，而是形成一套从数据整理、探索分析、模型训练、可解释性分析到交互式展示的完整研究型系统。

本项目的价值分为两层：

1. 理论价值

- 面向旅游地理学、时空序列预测与多源数据融合场景，研究气象、节假日、游客量等多因素对客流波动的共同作用。
- 通过多模型对比与解释性分析，探索非线性、多源驱动的旅游客流预测方法。

2. 应用价值

- 为荔波县旅游管理、节假日预警、景区资源调度提供数据支撑。
- 为后续同类旅游目的地的客流预测系统提供可复用的原型与技术路线。

项目当前采用“研究系统 + 工程原型并行”的方式推进：

- 研究系统：以数据分析、多模型实验、SHAP 解释和交互式研究看板为主。
- 工程原型：保留现有 FastAPI + MySQL + Vue 前后端链路，作为工程化演示基础。

### 二、研究思路与总体路线

本项目遵循“问题驱动 -> 数据驱动 -> 模型驱动 -> 应用落地”的总体思路：

1. 问题驱动

- 明确荔波县客流在普通工作日、周末、节假日和调休场景下的变化特征。
- 识别气象、时间、节假日等因素对客流的主要影响。

2. 数据驱动

- 基于现有 Excel/CSV 历史数据构建统一日级数据集。
- 对现有数据中的游客量、天气、节假日等字段进行清洗、标准化与时间对齐。
- 为后续门票销售、社交媒体、事件数据预留扩展接口与标准字段。

3. 模型驱动

- 先完成描述性统计、相关性分析和可解释性分析。
- 再进行特征工程与多模型实验，形成可比较、可复现的评估结果。

4. 应用落地

- 通过交互式仪表盘展示数据分布、预测结果、模型效果和解释性分析结果。
- 输出研究结论、管理建议和可答辩展示材料。

### 三、技术路线

本项目的目标技术路线与研究文本保持一致：

多源数据获取/导入 -> 清洗与时空对齐 -> 描述性统计 -> SHAP 可解释性分析 -> 特征工程 -> 多模型训练与融合 -> Streamlit + Plotly 交互式仪表盘 -> 结果解释与部署

结合当前仓库状态，开发落地分为两条线：

1. 研究主线

- 数据处理：Pandas、NumPy
- 统计分析：Pandas、SciPy、statsmodels
- 机器学习：scikit-learn、XGBoost、LightGBM
- 时间序列：ARIMA、Prophet
- 深度学习：LSTM、Temporal Fusion Transformer
- 可解释性：SHAP
- 可视化与展示：Streamlit + Plotly

2. 工程原型线

- 后端：FastAPI + SQLAlchemy + MySQL
- 前端：Vue3 + Vite + Element Plus + ECharts
- 报表导出：CSV / PNG

说明：

- 研究主线用于对齐开题/中期/答辩文本。
- 工程原型线保留当前已实现模块，便于系统联调、接口演示和数据管理。
- 两条线共享同一份基础数据，但展示和实验目标不同。

### 五、系统模块设计

#### 1. 数据导入与清洗模块

- 支持 Excel/CSV 导入
- 执行日期解析、数值转换、缺失值处理、重复数据覆盖
- 对日期属性进行统一标准化，包括工作日、周末、节假日、调休上班

#### 2. 探索分析模块

- 输出描述性统计结果
- 展示不同日期类型下的客流分布
- 提供相关性分析与趋势图

#### 3. 特征工程模块

- 生成时间特征
- 生成滞后特征与滚动统计特征
- 生成节假日类型、节前/节后窗口等特征
- 为树模型、时间序列模型和深度学习模型提供统一输入

#### 4. 多模型实验模块

模型实验采用分层推进方式：

1. 基线模型

- ARIMA
- Prophet

2. 机器学习模型

- XGBoost
- LightGBM

3. 深度学习模型

- LSTM
- Temporal Fusion Transformer

4. 融合策略

- 对多个模型输出统一评估表
- 支持简单加权平均、择优选择或对比展示

#### 5. 可解释性分析模块

- 使用 SHAP 分析树模型的重要特征
- 提供全局特征重要性图
- 提供样本级解释结果或示例

#### 6. 可视化展示模块

研究展示主界面采用：

- Streamlit
- Plotly

展示内容包括：

- 数据概览
- 时间趋势图
- 节假日与工作日对比
- 模型指标对比
- 实际值与预测值对比
- SHAP 解释图
